## MVP 定义

每个模块需要满足：有数据来源、可保存、可查看、可筛选/按账号隔离、可下钻详情、有动作/状态闭环、有异常说明、有验收或测试。

## 本次自动化验收

- `npm test -- --run`：通过，5 个测试文件，34 条测试；已覆盖 13 个模块每个至少 1 条主流程 MVP 测试，并补充岗位、素材、线索的 GUI 创建/更新/删除闭环。
- `npm run build`：通过，TypeScript 与 Vite 生产构建成功。

## 账号/角色隔离验收点

- `/api/data` 读取已按登录账号过滤可见岗位、内容、账号、素材、线索、落地页、复盘行动项、通知和审计日志。
- `/api/data` 保存时，管理员/招聘运营/招聘负责人保留全量写入；普通账号只合并自己可见范围内的数据，避免覆盖其他账号数据。
- 数据分析接口继续复用同一套隔离规则，保证列表、详情和指标下钻口径一致。

## 异常说明验收点

- 本地 API 未登录：返回 401 Unauthorized。
- 数据读取失败：返回数据存储不可用提示。
- 数据保存失败：返回请求格式或存储错误原因。
- 素材必填缺失：页面提示“保存失败：素材名称为必填项，请补充后再保存。”
- 工作台任务详情：展示空数据、权限不足或源数据缺失时的处理说明。
- 平台/API 失败：模型测试、集成同步、导入预检和发布检查会输出失败原因或阻断说明。
