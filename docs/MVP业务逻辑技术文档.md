# 招聘运营助手 MVP 业务逻辑技术文档

## 1. 文档目标

本文档定义招聘运营助手的 MVP 业务实现口径，粒度下钻到业务对象、动作事件、状态变化、字段依赖、数据关系、指标公式、时间范围和验收标准。

本文档优先级高于页面功能描述。页面只是承载业务动作，系统实现必须先满足本文档中的对象、事件、状态和指标计算规则。

## 2. MVP 最小单元定义

MVP 最小单元不是一个页面，也不是一个按钮，而是：

`业务对象 + 动作事件 + 上游依赖 + 状态变化 + 必要字段 + 可计算指标`

任何功能进入 MVP 前必须回答：

| 判断项 | MVP 要求 |
| --- | --- |
| 业务对象 | 必须归属到岗位、内容、发布记录、平台指标、招聘入口、北森回流、复盘动作之一 |
| 动作事件 | 必须是可审计的业务动作，例如创建、提交、审核、发布、导入、归因、复盘 |
| 上游依赖 | 必须说明动作发生前依赖哪些对象和字段 |
| 状态变化 | 动作完成后必须产生明确状态变化或数据写入 |
| 必要字段 | 必须列出最小字段集合，缺字段时动作是否允许继续 |
| 可计算指标 | 动作必须支撑至少一个核心指标或数据质量指标 |

不满足以上条件的能力不进入 MVP，可作为二期增强或隐藏配置。

## 3. 核心业务对象

| 对象 | 技术对象 | MVP 业务定义 | 关键主键 |
| --- | --- | --- | --- |
| 岗位 | `JobNeed` | 内容生产和候选人归因的源头 | `jobId` |
| 内容 | `ContentTask` | 围绕岗位在平台上发布的招聘内容 | `contentId` |
| 发布记录 | 当前并入 `ContentTask`，后续可独立为 `PublishRecord` | 内容实际发布到平台和账号的事实记录 | `publishId/contentId` |
| 平台指标 | `ContentTask.metrics` | 小红书等平台后台回收的内容表现数据 | `contentId + metricDate/importedAt` |
| 招聘入口 | `RecruitmentEntry`、`LandingPage` | 候选人从内容进入投递链路的入口 | `entryId/trackingCode` |
| 北森回流 | `BeisenResult` | 候选人投递和招聘阶段结果 | `candidateCode + jobId` |
| 复盘动作 | `ReportAction`、`TaskItem` | 系统根据数据判断生成的下一步动作 | `actionId/taskId` |

## 4. 业务主链路

系统的真实业务链路为：

`岗位 -> 内容 -> 发布 -> 平台数据 -> 招聘入口数据 -> 北森回流 -> 归因 -> 复盘动作`

这条链路有三条并行流程汇合：

| 流程 | 目标 | 汇合点 |
| --- | --- | --- |
| 内容生产流 | 决定发什么、谁审核、什么时候发 | 发布记录 |
| 数据回收流 | 判断内容是否被看见、观看、互动 | 内容效果 |
| 招聘转化流 | 判断内容是否带来真实候选人 | 北森回流与归因 |

三条流程最终都汇入复盘动作。复盘动作必须能反向定位到内容、岗位、平台、账号或入口。

## 5. MVP 动作事件定义

| 事件 ID | 动作事件 | 操作角色 | 上游依赖 | 必填字段 | 状态变化 | 产生数据 | 支撑指标 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `job.create` | 创建岗位 | 招聘运营/招聘专员 | 无 | 岗位名称、岗位族群、城市、JD、候选人画像、岗位卖点、招聘入口 | 岗位=招聘中 | 岗位基础数据 | 岗位完整率、内容可生产数 |
| `content.create` | 创建内容 | 招聘运营 | 岗位已存在 | `jobId`、平台、账号、标题、正文、CTA、入口 | 内容=草稿 | 内容草稿 | 内容任务数 |
| `content.submitReview` | 提交审核 | 招聘运营 | 内容字段完整 | 标题、正文、CTA、入口、账号、岗位 | 草稿/驳回修改 -> 待审核 | 审核任务 | 待审核数、字段完整率 |
| `content.reviewPass` | 审核通过 | 招聘负责人/审核人 | 内容=待审核，风险可接受 | 审核结果、审核人 | 待审核 -> 待发布 | 审核记录 | 审核通过率、待发布数 |
| `content.reviewReject` | 审核驳回 | 招聘负责人/审核人 | 内容=待审核 | 驳回原因 | 待审核 -> 驳回修改 | 审核记录 | 驳回率、风险内容数 |
| `publish.schedule` | 设置排期 | 招聘运营 | 内容=待发布，账号可用 | 发布时间、平台账号 | 状态不变 | 排期时间 | 待发布数、排期覆盖 |
| `publish.markPublished` | 标记发布 | 招聘运营 | 内容=待发布，平台链接存在 | 发布时间、平台链接 | 待发布 -> 数据回收中 | 发布事实 | 内容发布数、按时发布率 |
| `metrics.importPlatform` | 导入平台指标 | 招聘运营/系统同步 | 内容已发布且可匹配 | `contentId` 或标题、指标日期、平台指标 | 数据更新 | 小红书指标 | 曝光数、观看数、互动数 |
| `entry.collectClick` | 回收入口点击 | 系统埋点/导入 | 入口链接存在 | `entryId` 或 `trackingCode`、点击时间 | 数据更新 | 入口点击 | 招聘入口点击、观看到点击率 |
| `beisen.importStage` | 导入北森阶段 | 招聘运营/系统同步 | 北森数据可导出 | `candidateCode`、`jobId`、阶段、导入时间 | 数据更新 | 候选人阶段 | 投递数、有效简历数、面试数 |
| `attribution.match` | 归因匹配 | 系统 | 平台、入口、北森数据存在 | `contentId/jobId/trackingCode/candidateCode` 至少一类 | 归因完成/无法归因 | 归因关系 | 归因率、未归因数 |
| `review.generateAction` | 生成复盘动作 | 系统/招聘运营 | 至少有发布事实和一类回收数据 | 关联对象、动作类型、负责人、截止时间 | 内容可进入已复盘 | 行动项 | 复盘动作完成率 |

## 6. 状态机定义

### 6.1 内容状态机

| 状态 | 进入条件 | 允许动作 | 退出条件 | 阻断规则 |
| --- | --- | --- | --- | --- |
| 草稿 | 创建内容 | 编辑、提交审核 | 必要字段完整 | 缺岗位、缺入口、缺 CTA 时不能提交审核 |
| 待审核 | 提交审核 | 审核通过、驳回 | 审核结果产生 | 高风险未处理不能通过 |
| 驳回修改 | 审核驳回 | 编辑、重新提交 | 字段和风险重新满足要求 | 无驳回原因不允许驳回 |
| 待发布 | 审核通过 | 设置排期、标记发布 | 实际发布完成 | 缺账号、缺入口、缺平台链接不允许标记发布 |
| 数据回收中 | 标记发布 | 导入平台数据、导入入口点击、导入北森回流、生成复盘 | 达到复盘条件 | 无发布事实不允许导入为内容效果 |
| 已复盘 | 生成复盘动作 | 查看结论、复用模板 | 无 | 无复盘动作或结论不允许进入 |

### 6.2 数据完整状态

| 状态 | 判断条件 | 系统动作 |
| --- | --- | --- |
| 缺平台数据 | 内容已发布，但 `views + likes + comments + saves + shares = 0` | 首页生成数据待回收任务 |
| 缺入口数据 | 内容已发布，但招聘入口点击为 0 或入口未配置 | 提醒检查入口、短链或埋点 |
| 缺北森回流 | 有招聘入口点击，但无投递或有效简历 | 提醒导入北森结果或检查 trackingCode |
| 已完成回收 | 内容已发布，且平台数据、入口数据、北森数据至少有一类完成归因 | 允许进入复盘 |
| 无法归因 | 入口或北森数据无法匹配内容/岗位 | 进入数据质量问题 |

## 8. 数据关系和归因优先级

### 8.1 主关系

| 关系 | 基数 | 说明 |
| --- | --- | --- |
| 岗位 -> 内容 | 1:N | 一条内容必须归属一个岗位或岗位族群 |
| 内容 -> 发布记录 | 1:N | MVP 当前近似为 1:1，后续支持一稿多发 |
| 内容 -> 平台指标 | 1:N | 同一内容可多日期、多批次导入指标 |
| 内容 -> 招聘入口 | 1:N | 一条内容可绑定一个或多个入口 |
| 招聘入口 -> 点击 | 1:N | 点击按时间发生 |
| 候选人 -> 北森阶段 | 1:N | 同一候选人多阶段取最高阶段 |
| 复盘动作 -> 业务对象 | N:1 | 动作必须绑定内容、岗位、平台、账号或入口 |

### 8.2 归因优先级

北森回流和入口点击按以下优先级归因：

1. `sourceContentId` 直接匹配内容。
2. `trackingCode` 匹配招聘入口，再通过入口匹配内容和岗位。
3. `jobId + sourcePlatform` 匹配岗位和平台。
4. 仅有 `sourcePlatform` 时只能进入平台归因。
5. 以上均失败时进入无法归因数据。

同一候选人按 `candidateCode + jobId` 去重，阶段取最高阶段。

## 9. 指标计算口径

### 9.1 核心指标

| 指标 | 公式 | 分母为 0 时 | 时间字段 |
| --- | --- | --- | --- |
| 内容发布数 | `count(content.status in [数据回收中, 已复盘] or publishedAt exists)` | 0 | `publishedAt` |
| 曝光数 | `sum(metrics.impressions)` | 0 | `metricDate/importedAt` |
| 观看数 | `sum(metrics.views)` | 0 | `metricDate/importedAt` |
| 互动数 | `sum(likes + comments + saves + shares)` | 0 | `metricDate/importedAt` |
| 互动率 | `互动数 / 观看数` | 0% | 同观看数 |
| 招聘入口点击 | `sum(content.metrics.clicks or entry.clicks)`，同源需去重 | 0 | 点击发生时间 |
| 观看到入口点击率 | `招聘入口点击 / 观看数` | 0% | 同周期 |
| 投递数 | `count(bestStage(candidateCode + jobId) >= 已投递)` | 0 | `importedAt/stageChangedAt` |
| 有效简历数 | `count(bestStage(candidateCode + jobId) >= 有效简历)` | 0 | `importedAt/stageChangedAt` |
| 面试数 | `count(bestStage(candidateCode + jobId) >= 已约面)` | 0 | `importedAt/stageChangedAt` |
| Offer 数 | `count(bestStage(candidateCode + jobId) >= Offer)` | 0 | `importedAt/stageChangedAt` |
| 入职数 | `count(bestStage(candidateCode + jobId) = 已入职)` | 0 | `importedAt/stageChangedAt` |
| 点击到投递率 | `投递数 / 招聘入口点击` | 0% | 同周期 |
| 投递到有效率 | `有效简历数 / 投递数` | 0% | 同周期 |
| 数据回收完整率 | `已完成回收内容数 / 已发布内容数` | 0% | `publishedAt` |

### 9.2 阶段排序

北森阶段按如下顺序取最高阶段：

`已投递 < 有效简历 < 初筛通过 < 已约面 < 已面试 < Offer < 已入职`

例如同一 `candidateCode + jobId` 同时出现已投递、有效简历、已约面，只计为一名已约面候选人，同时也计入投递数和有效简历数。

### 9.3 数据完整状态公式

| 数据状态 | 公式 |
| --- | --- |
| 缺平台数据 | `publishedAt exists and views + likes + comments + saves + shares = 0` |
| 缺入口数据 | `publishedAt exists and clicks = 0` |
| 缺北森回流 | `clicks > 0 and attributedApplications = 0` |
| 已完成回收 | `publishedAt exists and (platformMetricsExists or clicks > 0 or attributedApplications > 0)` |
| 无法归因 | `beisenResult/sourceClick exists and no matched contentId/jobId/platform` |

## 10. 时间范围规则

指标默认不使用历史所有数据，必须带时间范围。

| 场景 | 默认时间范围 | 可配置 |
| --- | --- | --- |
| 运营首页 | 今天 + 近 7 天 | 可配置 |
| 渠道数据总览 | 近 30 天 | 可配置 |
| 内容复盘 | 近 7 天 | 可配置 |
| 岗位分析 | 岗位招聘周期 | 可配置 |
| 账号健康 | 近 30 天 | 可配置 |
| 周报/月报 | 用户选择周期 | 必须配置 |

时间字段按分析目的选择：

| 分析目的 | 使用时间字段 |
| --- | --- |
| 内容是否按时发布 | `publishedAt` |
| 内容平台表现 | `metricDate/importedAt` |
| 候选人新增和阶段推进 | `beisenImportedAt/stageChangedAt` |
| 入口转化 | `clickedAt` |
| 复盘动作完成 | `completedAt` |

如果看“某批内容发布后的效果”，先用 `publishedAt` 选内容，再读取这些内容在观察窗口内的平台指标、入口点击和北森回流。

如果看“本周新增候选人”，使用北森 `importedAt/stageChangedAt`，不能只按内容发布时间过滤。

## 11. 复盘规则

| 条件 | 业务判断 | 系统动作 |
| --- | --- | --- |
| 曝光低且观看低 | 分发弱或标题封面弱 | 生成改标题、改封面、换发布时间动作 |
| 曝光高且观看低 | 封面点击弱 | 生成优化首图和标题动作 |
| 观看高且互动低 | 内容共鸣弱 | 生成补候选人痛点、真实场景动作 |
| 互动高且招聘入口点击低 | CTA 或入口弱 | 生成强化 CTA、入口位置、岗位信息动作 |
| 招聘入口点击高且投递低 | 投递路径弱 | 生成检查北森链接、JD、投递门槛动作 |
| 投递高且有效低 | 人群不准 | 生成调整候选人画像和渠道表达动作 |
| 有效高且入口点击高 | 内容结构有效 | 生成复用内容模板动作 |

每条复盘动作必须包含：

| 字段 | 说明 |
| --- | --- |
| `targetType` | 内容、岗位、平台、账号、入口 |
| `targetId` | 被复盘对象 ID |
| `reason` | 触发规则和证据 |
| `action` | 具体动作 |
| `owner` | 负责人 |
| `dueDate` | 截止时间 |
| `status` | 未开始、进行中、已完成 |

## 12. MVP 不做或隐藏的能力

| 能力 | MVP 处理方式 | 原因 |
| --- | --- | --- |
| 自动发布到小红书 | 隐藏或后置 | API 可用性不确定，且不是当前闭环必要条件 |
| 复杂素材资产库 | 保留基础上传和授权状态 | 不直接决定有效候选人 |
| 多级审批流 | 只保留通过/驳回 | MVP 先验证内容发布闭环 |
| AI 自动长报告 | 后置 | 先用规则复盘保证可解释 |
| 粉丝画像分析 | 下钻或隐藏 | 对有效简历贡献间接 |
| 高级多触点归因模型 | 后置 | 先用 `contentId/trackingCode/candidateCode` 直接归因 |

## 13. 技术实现补充建议

### 13.1 需要补充的结构化字段

当前系统已有内容、岗位、指标和北森回流基础对象，但为了让业务逻辑更硬，需要补齐：

| 字段/对象 | 建议 |
| --- | --- |
| `ContentTask.cta` | 从正文里拆出结构化 CTA，提交审核和复盘都依赖它 |
| `ContentTask.entryId` | 内容必须绑定招聘入口，避免点击无法归因 |
| `ContentTask.platformUrl` | 标记发布时必须填写平台链接 |
| `MetricRecord.metricDate` | 平台指标应支持按日期统计，而不是只覆盖内容总值 |
| `EntryClick.clickedAt` | 入口点击应支持时间范围 |
| `BeisenResult.stageChangedAt` | 北森阶段最好保留阶段发生时间，导入时间只能作为 fallback |
| `AttributionRecord` | 建议二期独立保存归因结果和归因依据 |

### 13.2 导入幂等要求

平台指标、入口点击、北森回流都必须支持重复导入不重复计数。

| 数据 | 幂等键 |
| --- | --- |
| 平台指标 | `contentId + metricDate + platform + sourceBatchId` |
| 入口点击 | `entryId/trackingCode + clickedAt + visitorId/eventId` |
| 北森回流 | `candidateCode + jobId + stage` |
| 复盘动作 | `targetType + targetId + ruleId + periodStart + periodEnd` |

### 13.3 数据口径版本

指标公式应带 `metricSchemaVersion`。当前建议为 `xhs-mvp-v1`。

当后续小红书 API 或字段口径变化时，只新增版本，不直接覆盖旧公式，否则历史报表不可复算。

## 14. MVP 验收用例

| 用例 | 输入 | 预期结果 |
| --- | --- | --- |
| 岗位到内容 | 创建完整岗位后创建内容 | 内容可保存为草稿并绑定 `jobId` |
| 缺入口阻断 | 内容无 CTA 或入口时提交审核 | 系统阻止提交或生成缺字段问题 |
| 审核通过 | 待审核内容通过审核 | 状态变为待发布 |
| 发布事实 | 待发布内容填写平台链接并标记发布 | 状态变为数据回收中，产生 `publishedAt` |
| 平台指标导入 | 导入小红书曝光数、观看数、互动 | 内容指标更新，渠道数据可计算观看和互动 |
| 点击归因 | 导入招聘入口点击 | 招聘入口点击增加，可计算观看到点击率 |
| 北森去重 | 同一候选人多阶段导入 | 只计一名候选人，取最高阶段 |
| 复盘动作 | 观看高但点击低 | 生成强化 CTA/入口动作 |
| 时间范围 | 选择近 7 天和近 30 天 | 指标按对应时间字段重新计算 |
| 无法归因 | 北森数据无内容、岗位、平台 | 进入数据质量问题，不计入内容效果 |

## 15. 下一步落地顺序

1. 补齐结构化字段：`cta`、`entryId`、`platformUrl`、指标日期。
2. 把内容状态推进改为条件驱动，阻止无岗位、无入口、无审核的发布动作。
3. 把平台指标从覆盖式 `ContentTask.metrics` 升级为可按日期导入的指标明细。
4. 实现归因服务：统一处理 `contentId/trackingCode/candidateCode`。
5. 实现时间范围配置：所有渠道数据、复盘、首页指标共用同一套查询参数。
6. 把复盘动作从文本建议升级为可跟踪的 `ReportAction/TaskItem`。
