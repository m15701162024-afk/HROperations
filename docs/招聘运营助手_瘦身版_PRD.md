# 招聘运营助手瘦身版 PRD

## 1. 文档信息

- 产品名称：招聘运营助手
- 版本：V1.3 瘦身版
- 文档类型：产品需求文档
- 核心方向：真实平台 API 对接、真实账号同步、内容运营闭环、数据归因分析
- 面向对象：招聘专员、新媒体运营、招聘负责人、系统管理员、研发团队
- 当前背景：系统已具备较多模块，但存在演示台账、人工状态维护、重复入口和低频配置过多的问题，需要收敛为可真实落地的 MVP。

## 2. 冗余定义

本项目中的“冗余”定义为：对真实招聘运营闭环没有必要贡献，或可以由平台 API、北森 API、系统规则自动产生，却仍要求用户通过独立入口、表单、状态或台账手工维护的功能。

判断标准：

| 类型 | 定义 | 处理原则 |
| --- | --- | --- |
| 数据源冗余 | 真实平台/API 可提供的数据，却仍要求手填、CSV 导入或人工切状态 | 删除手填入口，改为 API 同步 |
| 入口冗余 | 同一业务动作分散在多个一级模块 | 合并到主流程模块 |
| 流程冗余 | 不是招聘运营核心链路，只为演示、验收或台账存在 | 删除或降级为管理员工具 |
| 角色冗余 | 日常运营用户接触低频系统配置 | 收进系统配置，仅管理员可见 |
| 结果冗余 | 只是换一种展示已有分析，没有新增决策价值 | 合并为页签或导出能力 |

## 3. 产品目标

### 3.1 业务目标

1. 让系统从“功能大全”收敛为“招聘内容投放 + 真实平台数据 + 北森归因”的运营工具。
2. 平台账号必须来自真实 API 同步，不再允许虚拟账号或人工授权状态。
3. 降低招聘专员和运营人员的日常操作路径，把高频动作集中到 4-5 个页面。
4. 将分析和复盘建立在真实内容、真实账号、真实线索、真实指标之上。

### 3.2 产品目标

1. 一级导航从 12 个左右收敛到 6-7 个。
2. 删除账号手工维护、账号 CSV 导入、人工授权状态、独立 AI 工作台、独立复盘报告、独立选题库、独立排期日历等冗余入口。
3. 保留必要的管理员配置，但不暴露给日常运营流。
4. 用 API 集成作为平台账号、平台指标、北森线索同步的唯一真实入口。

## 4. 用户角色

| 角色 | 核心诉求 | 可见模块 |
| --- | --- | --- |
| 招聘专员 | 创建岗位、查看内容、处理线索、跟进转北森 | 工作台、招聘需求、内容运营、线索池、数据分析 |
| 新媒体运营 | 生成内容、排期、发布、同步平台账号和指标 | 工作台、内容运营、账号与平台、数据分析 |
| 招聘负责人 | 看平台表现、岗位转化、线索质量、复盘结论 | 工作台、数据分析、线索池 |
| 系统管理员 | 配置登录账号、角色、API、字段映射、备份 | 系统配置、账号与平台 |

## 5. 瘦身后信息架构

### 5.1 保留一级模块

| 模块 | 定位 | 说明 |
| --- | --- | --- |
| 工作台 | 今日待办和关键指标入口 | 只展示真实数据派生任务和快捷入口 |
| 招聘需求 | 岗位源数据 | 内容和线索归因的业务源头 |
| 内容运营 | 内容生产、审核、排期、发布 | 合并原选题库和排期日历 |
| 账号与平台 | API 集成、真实账号同步、招聘入口 | 删除人工账号台账 |
| 线索池 | 平台线索、北森转入前置池 | 优先来自 API/落地页/北森回流 |
| 数据分析 | 平台、账号、内容、岗位、漏斗、复盘 | 合并原复盘报告 |
| 系统配置 | 角色、登录账号、规则、备份 | 仅管理员可见 |

### 5.2 删除或合并模块

| 原模块/能力 | 处理方式 | 原因 |
| --- | --- | --- |
| 选题库 | 合并到内容运营 | 选题是内容前置状态，不需要独立一级入口 |
| 排期日历 | 合并到内容运营 | 排期是内容状态的一部分 |
| 素材资产 | MVP 删除一级入口，仅保留内容附件/授权字段 | 独立素材库偏资产管理，当前不是主闭环 |
| 导入中心 | 删除一级入口，保留为系统配置中的异常补录工具 | 真实 API 对接后，导入只做兜底 |
| 复盘报告 | 合并到数据分析 | 复盘是分析结果的输出形态 |
| AI 工作台 | 删除一级入口 | 模型配置进系统配置，AI 能力嵌入内容/分析 |
| 账号健康度独立概念 | 保留为数据分析指标 | 不再作为账号管理台账 |
| 落地页生成 | MVP 删除或后置 | 若投递统一走北森/官网，落地页不是必需 |
| 企业微信/飞书摘要 | 后置 | 非主闭环 |
| BI 集成 | 后置 | 与数据分析重叠，等真实 BI 场景明确后再做 |

## 6. 核心业务流程

### 6.1 平台账号真实对接流程

1. 管理员进入账号与平台。
2. 配置平台 API：接口地址、Token/API Key、鉴权方式、扩展 JSON、字段映射。
3. 点击测试连接。
4. 点击同步账号。
5. 系统从平台接口读取账号数组，写入真实账号缓存。
6. 内容运营和数据分析只能选择这些同步账号。

约束：

- 不允许手工新增平台账号。
- 不允许手工切换账号授权状态。
- 不允许通过 CSV 导入账号。
- 账号状态来自 API 同步结果或连接结果。

### 6.2 内容运营流程

1. 从招聘需求选择岗位。
2. 生成或手写平台内容。
3. 选择真实平台账号。
4. 进行风险扫描和质量评分。
5. 设置排期时间。
6. 发布后进入数据待回收状态。
7. 通过平台 API 拉取指标。
8. 数据分析查看转化表现。

### 6.3 线索归因流程

1. 候选人线索来自平台、落地页、手工补录或北森回流。
2. 线索绑定平台、真实账号、内容、岗位。
3. 系统识别重复线索。
4. 待转入北森的线索通过北森 API 同步。
5. 北森结果回流后进入数据分析漏斗。

### 6.4 数据分析和复盘流程

1. 平台 API 拉取曝光、互动、点击等指标。
2. 北森 API 或导入回流投递、面试、Offer、入职。
3. 数据分析按平台、账号、内容、岗位、漏斗展示。
4. 复盘报告作为数据分析页签生成，不再独立成模块。
5. 支持导出 Markdown/HTML/CSV。

## 7. 功能需求

## 7.1 工作台

### 范围

保留：

- 今日待办
- 数据待回收
- 待审核内容
- 待跟进线索
- API 同步失败提醒
- 核心指标卡

删除：

- 目标管理表单
- 过多通知台账
- MVP 验收型提示

### 验收标准

- 无真实数据时显示空状态，不生成演示任务。
- 每个待办能跳转到对应业务对象。
- API 同步失败必须在工作台可见。

## 7.2 招聘需求

### 范围

保留：

- 岗位新增、编辑、关闭
- JD、候选人画像、岗位卖点
- 北森岗位 URL、官网 URL
- 目标平台

删除或后置：

- 与内容无关的复杂岗位运营字段

### 验收标准

- 内容必须能关联岗位。
- 线索必须能关联岗位。
- 数据分析可按岗位聚合。

## 7.3 内容运营

### 范围

合并原能力：

- 内容创建
- AI 内容生成
- 风险扫描
- 内容质量评分
- 审核状态推进
- 选题状态
- 排期时间
- 内容日历视图

删除或后置：

- 独立选题库一级入口
- 独立排期日历一级入口
- 复杂版本对比 UI 可后置

### 验收标准

- 内容只能绑定真实 API 同步账号。
- 内容可从草稿推进到发布和数据回收。
- 排期冲突可以提示。
- 内容指标来自平台 API 或管理员异常补录。

## 7.4 账号与平台

### 范围

保留：

- 平台 API 集成配置
- 测试连接
- 同步真实账号
- 拉取平台指标
- 北森线索同步
- 招聘入口配置
- 同步运行记录

删除：

- 手工新增平台账号
- 手工编辑平台账号
- 手工授权状态
- 发布权限角色
- 账号定位
- 账号 CSV 导入
- 落地页生成和表单留资
- 企业微信/飞书摘要
- BI 集成

### 验收标准

- 平台账号列表只能展示 API 同步结果。
- 删除 API 集成时，应提示该集成下账号缓存会失效或需要清理。
- API 同步失败应记录失败原因。
- 平台指标拉取可写回内容指标。

## 7.5 线索池

### 范围

保留：

- 线索列表
- 线索去重
- 线索跟进记录
- 线索绑定岗位、内容、真实账号
- 转入北森
- 北森状态

收缩：

- 手工录入只作为补录能力，不作为主路径。
- CSV 导入只作为管理员异常补录。

### 验收标准

- 同联系方式线索提示重复。
- 线索可同步北森并回写状态。
- 数据分析能按线索来源统计转化。

## 7.6 数据分析

### 范围

保留并合并：

- 总览
- 平台分析
- 账号分析
- 内容分析
- 岗位分析
- 漏斗归因
- 数据质量问题
- 复盘报告生成和导出

删除：

- 单独复盘报告一级入口
- 与数据分析重复的报告列表页

### 验收标准

- 账号分析只使用真实 API 同步账号。
- 能定位内容未绑定真实账号、指标未同步、北森未回流等质量问题。
- 可生成复盘摘要并导出。

## 7.7 系统配置

### 范围

保留：

- 角色权限
- 本地登录账号
- API 字段映射
- 高风险规则
- 模型 API 配置
- 系统健康检查
- 备份

删除或隐藏：

- MVP 验收矩阵
- 插件规则
- 部署任务
- 独立 AI 试跑工作台
- 日常运营不需要看到的系统台账

### 验收标准

- 仅管理员可见。
- 非管理员不能配置登录账号和 API 密钥。
- 敏感密钥不在导出文件中明文暴露。

## 8. MVP 范围

### P0 必须交付

1. 一级导航瘦身。
2. 删除平台账号手工维护和账号导入。
3. 平台 API 同步真实账号。
4. 内容运营合并选题和排期。
5. 数据分析合并复盘报告。
6. 导入中心下线为一级入口。
7. 系统配置隐藏低频台账。

### P1 后续增强

1. OAuth 标准授权流程。
2. 北森正式字段映射。
3. 平台发布 API。
4. 飞书/企微通知。
5. BI 输出。

### 不做范围

- 不做虚拟账号。
- 不做人工授权状态。
- 不做完整素材资产管理。
- 不做独立 AI 控制台。
- 不做独立落地页建站器。

## 9. 成功指标

| 指标 | 目标 |
| --- | --- |
| 一级导航数量 | <= 7 |
| 平台账号来源 | 100% 来自 API 同步 |
| 内容绑定真实账号比例 | >= 95% |
| 手工导入入口 | 仅管理员可见 |
| 日常运营完成一次内容发布路径 | <= 4 个页面切换 |
| API 同步失败可见性 | 工作台和账号与平台均可见 |
