# 招聘运营助手二期增强 PRD

## 1. 文档信息

- 产品名称：招聘运营助手
- 版本：V2.0 增强版
- 文档类型：产品需求文档
- 面向对象：招聘团队、新媒体运营、技术负责人、管理层、研发/设计团队
- 背景版本：现有系统已覆盖岗位需求、内容运营、素材资产、账号与平台、导入中心、数据分析、复盘报告、AI 工作台、系统配置等主链路

## 2. 项目背景

当前系统已经具备招聘新媒体运营的基础闭环，但从真实招聘运营使用场景看，仍存在以下问题：

1. 日常使用缺少任务化工作台，用户打开系统后不能第一时间知道今天该处理什么。
2. 内容生产偏单次生成，缺少长期选题规划、内容质量评估和平台策略沉淀。
3. 候选人线索在进入北森前缺少轻量管理池，平台私信、评论、落地页线索难以归因和跟进。
4. 排期日历还不够像真实运营排班工具，缺少拖拽、冲突、频次目标和节点管理。
5. 账号管理缺少账号健康度，无法判断账号是否停更、跑偏或低效。
6. 复盘报告缺少自动生成和业务解释能力，管理层难以快速看懂结果。
7. 多人协作审核流程还不够细，HR、运营、技术负责人、合规之间缺少闭环。
8. 后续接入真实平台 API 和北森 OpenAPI 后，需要更强的数据容错、口径说明和失败处理。

## 3. 产品目标

### 3.1 业务目标

- 让招聘运营团队从“手工发内容”升级为“任务驱动 + 数据复盘 + 线索归因”的运营方式。
- 提升中高端技术岗位、稀缺岗位在新媒体平台的曝光、点击、投递和有效简历转化。
- 降低 HR、新媒体运营和技术负责人之间的沟通成本。
- 为后续接入北森、平台 API、企业微信/飞书、BI 提供清晰的数据和流程基础。

### 3.2 产品目标

- 形成可执行的今日任务中心。
- 建立选题、内容质量评分、排期、发布、回收、复盘的完整内容运营链路。
- 建立平台线索池，支持线索跟进、去重、转北森和归因。
- 建立账号健康度和平台策略分析。
- 让复盘报告可以自动生成、解释和导出。

## 6. 功能需求

## 6.1 P0-任务化工作台

### 6.1.1 需求说明

工作台从“指标展示”升级为“今日任务中心”，用户进入系统后可以直接看到待处理事项，并能跳转到对应模块处理。

### 6.1.2 任务类型

| 任务类型 | 触发条件 | 处理入口 |
| --- | --- | --- |
| 今日待发布 | 内容排期日期等于今天且状态未发布 | 内容运营/排期日历 |
| 今日待审核 | 内容状态为待专业审核、待品牌合规审核 | 内容详情 |
| 数据待回收 | 内容已发布超过配置天数但无指标 | 数据分析/导入中心 |
| 高风险内容待处理 | 内容风险等级为高且未审核通过 | 内容详情 |
| 素材授权即将过期 | 素材有效期小于等于 30 天 | 素材资产 |
| 线索待跟进 | 线索状态为待联系、已联系未转北森 | 线索池 |
| 审核超时 | 审核节点超过 SLA | 协作审核 |
| 账号停更提醒 | 账号超过配置天数未发布内容 | 账号健康度 |

### 6.1.3 页面能力

- 今日任务列表
- 按任务类型筛选
- 按负责人筛选
- 一键跳转处理
- 批量标记完成
- 任务逾期标识
- 任务数量汇总

### 6.1.4 验收标准

- 用户进入工作台即可看到待办任务。
- 所有任务均可点击跳转到对应业务对象。
- 没有真实数据时任务数量为 0，不展示演示数据。
- 任务完成后工作台数量实时变化。

## 6.2 P0-候选人线索池

### 6.2.1 需求说明

在候选人正式进入北森前，系统需要管理来自小红书、脉脉、B站、公众号、抖音、知乎、技术社区、落地页等渠道的候选人线索。

### 6.2.2 线索来源

- 平台私信
- 评论区意向
- 落地页表单
- HR 手动录入
- CSV 导入
- 浏览器插件采集
- 平台 API 拉取

### 6.2.3 线索字段

| 字段 | 说明 |
| --- | --- |
| 姓名/昵称 | 候选人姓名或平台昵称 |
| 联系方式 | 手机、邮箱、微信等 |
| 来源平台 | 小红书、脉脉、B站等 |
| 来源账号 | 关联平台账号 |
| 来源内容 | 关联内容任务 |
| 意向岗位 | 关联岗位需求 |
| 当前阶段 | 待联系、已联系、已转北森、无效、暂不合适 |
| 跟进人 | HR 或运营负责人 |
| 跟进记录 | 沟通记录 |
| 重复标识 | 是否疑似重复 |
| 北森状态 | 待转入、已转入、转入失败 |

### 6.2.4 核心能力

- 新增线索
- CSV 批量导入线索
- 线索详情下钻
- 跟进记录
- 重复线索识别
- 批量分配跟进人
- 转入北森前置池
- 转入北森结果回写
- 按平台、岗位、内容归因

### 6.2.5 验收标准

- 用户可以新增、编辑、删除、筛选线索。
- 同手机号/邮箱/微信的线索会提示重复。
- 线索可关联岗位、内容、账号和平台。
- 线索可从待转入变为已转入北森。
- 数据分析可统计线索来源和转化。

## 6.3 P0-内容质量评分

### 6.3.1 需求说明

系统需要在内容发布前给出质量评分，帮助 HR 和运营判断内容是否适合发布。

### 6.3.2 评分维度

| 维度 | 分值 | 说明 |
| --- | --- | --- |
| 标题吸引力 | 0-20 | 是否具体、有候选人视角、有平台感 |
| 候选人关注点覆盖 | 0-20 | 是否覆盖薪酬、稳定性、挑战、氛围、成长等 |
| 岗位卖点覆盖 | 0-20 | 是否体现岗位价值、团队、技术挑战 |
| 平台适配度 | 0-15 | 是否符合小红书/脉脉/B站等平台表达 |
| CTA 明确度 | 0-10 | 是否有明确投递/私信/查看岗位入口 |
| 合规风险 | 0-15 | 是否存在高风险表达，风险越高扣分越多 |

### 6.3.3 页面能力

- 内容详情显示总分
- 显示各维度得分
- 显示低分原因
- AI 给出修改建议
- 支持重新评分
- 支持评分历史记录

### 6.3.4 验收标准

- 每条内容可生成质量分。
- 高风险内容不得获得高合规分。
- 评分结果可保存并展示历史。
- 低于阈值的内容发布时给出提醒。

## 6.4 P0-排期日历增强

### 6.4.1 需求说明

排期日历需要从列表式展示升级为真实运营排班工具。

### 6.4.2 核心能力

- 周视图
- 月视图
- 按平台筛选
- 按账号筛选
- 按负责人筛选
- 内容详情弹窗
- 修改发布时间
- 发布冲突提醒
- 平台频次目标提醒
- 节假日/招聘节点标记

### 6.4.3 冲突规则

| 冲突类型 | 判断逻辑 |
| --- | --- |
| 同账号同日过载 | 同账号同日发布数超过配置阈值 |
| 平台频次不足 | 周发布数量低于平台目标 |
| 高风险未审发布 | 高风险内容未完成审核进入发布日 |
| 入口未配置 | 内容对应平台无招聘入口 |
| 素材未授权 | 内容关联素材未授权 |

### 6.4.4 验收标准

- 用户可以在周/月视图查看内容排期。
- 点击日历内容可打开详情。
- 修改日期后内容排期同步更新。
- 冲突内容有明显标识和说明。

## 6.9 P2-智能数据解释

### 6.9.1 需求说明

在数据分析页给出业务解释，而不是只展示指标。

### 6.9.2 解释示例

- 小红书曝光高但投递低，可能是 CTA 不清晰或岗位入口不明显。
- 脉脉点击率高但有效简历低，可能是岗位定位与受众不匹配。
- B站内容互动高但点击低，适合做品牌长期影响，不适合作为短期投递主渠道。

## 6.10 P2-平台策略建议

### 6.10.1 需求说明

### 6.10.2 建议维度

- 发布频次建议
- 标题风格建议
- 账号建议
- 是否需要技术负责人出镜/署名

## 9. 非功能需求

- 所有数据默认真实数据，未录入时显示 0 或空状态。
- 所有新增模块需支持本地 JSON 存储，并预留 API 接入能力。
- 页面在 1366px、1440px、1920px 和移动端宽度下无横向溢出。
- 关键操作需要写入审计日志。
- 导入、同步、AI 调用失败时需要有明确错误提示。
- 保持当前部署地址配置不被代码改动影响。
